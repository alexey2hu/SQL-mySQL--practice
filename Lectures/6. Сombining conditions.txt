# Комбинирование условий

/*
** Логические операторы: ('AND', 'OR', 'NOT')
** Работают с уже готовыми логическими значениями
** Имеют больший приоритет чем у остальных операторов
** 
** AND  	 - (И)		 (Условие истинно, если оба условия истинны.)
** OR   	 - (ИЛИ)	 (Условие истинно, если хотя бы одно из условий истинно.)
** NOT		 - (Не)		 (Логическое отрицание. Инвертирует условие.)
*/

# AND, OR:
	/*
	** Аналог (&& и ||) из множества ЯП
	*/

	# Пример:
	SELECT *
	FROM customers
	WHERE first_name = 'Freddi'
		OR points > 1000 AND state = 'VA'

# NOT:
	/*
	** (TRUE = FALSE & FALSE = TRUE)
	** Полная инверсия, как единичных условий, так и итогового ответа в ()
	**
	** Может стоять перед любым условием
	*/

	# Пример:
	SELECT *
	FROM customers
	WHERE NOT (points < 2000 AND state = 'VA')

	/*
	** Аналогично
	** WHERE points >= 2000 OR state != 'VA'
	*/

/*
** Предикаты или Операторы сравнения: ('IN', 'LIKE', 'BETWEEN')
** Сами формируют логическое значение из нелогических данных (чисел, строк, дат).
** 
** IN()    - (В)		 (Проверяет, совпадает ли значение с одним из списка внутри IN)
** LIKE 	 - (Как)	 (Поиск по шаблону 'паттерну', работает с тестовыми полями)
** BETWEEN - (Между) (Проверяет, находится ли значение в диапазоне включительно.)
**
** Их разберем позже
*//
